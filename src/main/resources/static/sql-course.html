SQL & Databases Course

  1. Module 1: Introduction to Databases

    Definition: What is a Database?
Example: Relational vs NoSQL overview

  2. Module 2: SQL Basics

    Definition: Basic SQL commands
Example: CREATE TABLE users (id INT, name VARCHAR(50));

  3. Module 3: SELECT Queries

    Definition: Retrieve data from tables
Example: SELECT name FROM users;

  4. Module 4: WHERE Clause

    Definition: Filter rows with conditions
Example: SELECT * FROM users WHERE id = 1;

  5. Module 5: UPDATE & DELETE

    Definition: Modify or remove data
Example: UPDATE users SET name='Alice' WHERE id=1;
         DELETE FROM users WHERE id=1;

  6. Module 6: JOINs (INNER, LEFT, RIGHT)

    Definition: Combine rows from multiple tables
Example: SELECT * FROM orders INNER JOIN customers ON orders.cid = customers.id;

  7. Module 7: GROUP BY & Aggregates

    Definition: Group rows and apply functions
Example: SELECT dept, COUNT(*) FROM employees GROUP BY dept;

  8. Module 8: Indexes & Performance

    Definition: Speed up queries with indexes
Example: CREATE INDEX idx_name ON users(name);

  9. Module 9: Transactions & ACID

    Definition: Ensure reliable database operations
Example: BEGIN; UPDATE accounts SET balance=balance-100 WHERE id=1; COMMIT;

  10. Module 10: Constraints (PRIMARY, FOREIGN)

    Definition: Rules for data integrity
Example: PRIMARY KEY(id), FOREIGN KEY(dept_id) REFERENCES departments(id);

  11. Module 11: Views & Stored Procedures

    Definition: Virtual tables and reusable SQL blocks
Example: CREATE VIEW active_users AS SELECT * FROM users WHERE active=1;

  12. Module 12: Triggers

    Definition: Automatic actions on events
Example: CREATE TRIGGER log_update AFTER UPDATE ON users FOR EACH ROW ...

  13. Module 13: Normalization

    Definition: Organize data to reduce redundancy
Example: Splitting user info into separate tables

  14. Module 14: Relationships

    Definition: Define links between tables
Example: One-to-many relationship between customers and orders

  15. Module 15: Backup & Restore

    Definition: Save and recover database state
Example: mysqldump -u root dbname > backup.sql

  16. Module 16: Advanced Queries

    Definition: Complex SQL operations
Example: SELECT dept, AVG(salary) FROM employees WHERE salary>50000 GROUP BY dept;

  17. Module 17: Subqueries

    Definition: Nested queries inside another
Example: SELECT name FROM users WHERE id IN (SELECT user_id FROM orders);

  18. Module 18: SQL Functions

    Definition: Built-in operations on data
Example: SELECT UPPER(name), NOW() FROM users;

  19. Module 19: Practice Exercises

    Definition: Hands-on SQL problems
Example: Write queries to find top 5 highest salaries

  20. Module 20: Project: Student Database

    Definition: Final project applying all concepts
Example: Build a student management system with CRUD operations
